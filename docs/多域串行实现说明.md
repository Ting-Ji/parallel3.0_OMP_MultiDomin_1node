# 多域串行实现阶段说明

本文档记录到当前阶段为止，在最初单域 TDBEM 程序基础上已经完成的多域串行改造、验证结论和仍然保留的限制。当前结论只针对串行多域路径，不包含并行化、100 域完整验证、分段材料验证或复杂边界条件验证。

## 当前阶段结论

程序已经初步具备在 10 个共形子域、当前整块边界条件下进行正确串行计算的能力。

这个结论的依据是：

1. `DomainCount = 1` 时，新多域路径与旧单域路径结果完全一致。
   - `NStep = 2` 和 `NStep = 5` 的 `validation_state.csv` 对比结果均为 `max_abs = 0`。
   - 这说明新增多域入口没有破坏旧单域动态求解链路。

2. `DomainCount = 1` 的矩阵自检已作为硬门槛。
   - 新路径对 `MTS0`、`MTS[0]`、`MTS[1]`、`MGS[1]` 做矩阵-向量乘法等价检查。
   - 自检通过后才进入多域 GMRES 求解。

3. `DomainCount = 10`、同材料、`NStep = 5` 的串行验证通过。
   - GMRES 正常收敛。
   - 外边界与同离散单域基准最大绝对误差为 `9.045260e-10`。
   - 所有界面满足 `max |u_left - u_right| = 0` 和 `max |t_left + t_right| = 0`。

4. `DomainCount = 2` 的粗模型差异已被诊断定位。
   - 最大外边界差异出现在 step 4 左端 `ty`，数值为 `3.861794e-05`。
   - 但 `A*x-rhs` 残差达标，多域最大相对残差约为 `2.78e-12`。
   - RHS 分量差异很小，最大约为 `4.95e-09`。
   - 界面连续和平衡误差为 0。
   - 因此该差异目前不指向 GMRES 未收敛、RHS 历史项主导错误或界面条件失效，更像是粗 2 域模型与单域粗基准之间不可作为严格等价门槛的离散差异。

因此，当前可以把 10 域同材料长杆、当前 `BCID=123/456` 整块边界条件、串行 GMRES-mCCSR 路径作为已经初步跑通的阶段性成果。但不能把这个结论外推到所有边界条件、所有材料组合、100 域规模或并行求解。

## 基于旧单域版本保留的求解链路

本阶段没有重写边界元积分公式，也没有引入 ACA、远场近似、新压缩格式或新 GMRES 算法。多域代码尽量复用旧单域版本中已经验证过的链路：

- 当前步未知矩阵仍参考 `GetMatrixDynaUnknown0ij` 和 `UnknownSubMatrix`。
- 当前步已知矩阵仍参考 `GetMatrixDynaKnown0ij` 和 `knownSubMatrix`。
- 历史面力矩阵仍按旧逻辑形成 `T2 + T1`。
- 历史位移矩阵仍参考旧 `GetMatrixDynaUij` / `PthGetMatrixDynaSymUijNew` 语义。
- RHS 递推仍沿用旧公式：

```txt
B = MTS[0] * knownCurrent
B += MGS[j] * (T[k+1] + 2T[k] + T[k-1])
B -= MTS[j] * (U[k+1] + 2U[k] + U[k-1])
```

新增多域代码只承担三类工作：

- 控制每个子域内部的源点和边界单元循环范围。
- 通过全局自由度映射处理外边界未知量和界面共享自由度。
- 用块稀疏 builder 对重复 `(rowBlock, colBlock)` 贡献做累加。

## 已完成的主要代码修改

### 1. 新增多域模块

新增文件：

- `DBEM1/multidomain.h`
- `DBEM1/multidomain.cpp`

主要类型：

- `Domain`
- `InterfacePair`
- `MultiDomainModel`
- `DomainMaterialContext`
- `GlobalDofMap`
- `MultiDomainCCSRBuilder`
- `MultiDomainState`

这些结构将原来单一全局 `NodeNum/EleNum` 思路拆成：

- 子域拓扑。
- 子域局部边界单元。
- 全局方程行。
- 全局未知量块。
- 界面共享位移和反号面力。

### 2. 扩展边界单元信息

给 `DSquareElement` 增加了多域相关字段：

```cpp
int DomainID;
int LocalEleID;
int SurfaceType;
```

用途：

- `DomainID` 标记单元属于哪个子域。
- `LocalEleID` 标记单元在本子域内的局部编号。
- `SurfaceType` 区分外边界和内部界面。

### 3. 材料参数局部化

新增 `BuildDynaMat(...)`，避免复制旧 `StaticInit` 中材料和波速系数计算公式。

当前策略：

- 单域路径仍保留旧材料初始化行为。
- 多域路径为每个子域构造一套 `DynaMat`。
- 积分子块计算前按当前边界单元所属域切换材料上下文。

当前验证主要覆盖同材料多域。分段材料虽然已有数据结构支撑，但还未作为本阶段验收目标。

### 4. 多域输入与自动长杆划分

参数卡在旧格式之后可追加多域段：

```txt
DomainCount
domainId E v Rou
...
InterfaceCount
domainA eleA domainB eleB normalSign
...
```

兼容规则：

- 没有 `DomainCount` 时继续走旧单域路径。
- 有 `DomainCount` 时启用多域路径。
- 当前验证算例采用长杆按单元顺序划分子域。
- 界面单元共形，一一配对。

### 5. 全局自由度映射

当前多域未知量策略为：

- 外边界仍由 `BCID` 决定未知量。
  - `BCID=123`：给定位移，未知面力。
  - `BCID=456`：给定面力，未知位移。
- 内界面只保留一套界面位移 `uI` 和一套界面面力 `tI`。
  - A 侧：`uA = uI`，`tA = tI`
  - B 侧：`uB = uI`，`tB = -tI`

该映射保证：

- 界面位移自动连续。
- 界面面力自动平衡。
- 多个子域之间不会重复创建界面未知量。

### 6. 多域块稀疏装配器

新增 `MultiDomainCCSRBuilder`，用于替代直接追加块的旧思路。

核心原因：

- 多域装配中多个子域方程可能写入同一个全局自由度块。
- 旧 `WmatrixCCSR::assign` 更接近追加写入，不适合作为重复块累加器。

当前实现：

- 使用 `std::map<std::pair<long,long>, std::array<double,9>>` 聚合块。
- `AddBlock(...)` 自动对重复 `(rowBlock, colBlock)` 累加。
- `Build(...)` 输出现有 `CCSRMat`，继续供 `SparseMul` 和 `gmres` 使用。

### 7. 多域动态求解入口

新增入口：

```cpp
int DynaGMRESSolverMultiDomainCCSR(...)
```

该入口完成：

- 构建 `MTS0` 当前步未知矩阵。
- 构建 `MTS[0]` 当前步已知矩阵。
- 构建 `MTS[step] = T2 + T1` 历史面力矩阵。
- 构建 `MGS[step] = U` 历史位移矩阵。
- 使用 `MultiDomainState` 保存每步 `globalU/globalT/globalUnknown/globalKnown`。
- 沿用现有 `gmres(..., CCSRMat& A, ...)` 求解。
- 解后反写到每个边界单元的位移和面力状态。

### 8. 验证和诊断输出

新增环境变量：

```txt
DBEM_VALIDATION_OUTPUT=1
DBEM_RHS_DIAGNOSTIC=1
```

主要输出：

- `output/validation_state.csv`
  - 每步每个单元的域编号、边界类型、面心坐标、位移和面力。
- `output/validation_metrics.txt`
  - GMRES 状态、界面误差、非零块数、耗时等。
- `output/rhs_breakdown.csv`
  - RHS 分解：`knownRhs`、`historyG`、`historyT`、`totalRhs`、`Ax`、`residual`。
- `output/residual_probe.csv`
  - 每步 `A*x-rhs` 残差。
- `output/interface_probe.csv`
  - 每步每个界面两侧的 `uA/uB/tA/tB`。

验证脚本：

- `tools/run_multidomain_validation.py`

新增模式：

```bash
python tools/run_multidomain_validation.py --diagnose-two-domain
python tools/run_multidomain_validation.py --ten
```

诊断模式会生成：

- `state_outer_diff.csv`
- `rhs_outer_diff.csv`

用于按差异大小排序查看外边界状态差异和 RHS 分量差异。

## 当前限制

当前阶段仍有明确边界：

- 只确认串行多域路径。
- 暂不加入并行多域装配或并行多域求解。
- 暂不运行 100 域完整验证。
- 暂不把分段材料作为本阶段验收结果。
- 当前只验证整块 `BCID=123/456`。
- 暂不支持混合方向边界条件。
- 暂不引入 ACA、远场近似或额外压缩。
- 暂不接复杂几何树预处理器。

## 下一步建议

下一阶段建议继续保持务实顺序：

1. 将 10 域同材料、`BCID=123/456`、`NStep=5` 作为串行正确性基准。
2. 继续保留 1 域新旧路径等价检查作为硬门槛。
3. 对 2 域粗模型保留诊断，不把它作为严格外边界等价失败的阻断项。
4. 在 10 域基准稳定后，再做 100 域同材料验证。
5. 100 域同材料通过后，再做分段材料验证。
6. 串行结果闭环后，再考虑恢复或新增多域并行装配和求解。
